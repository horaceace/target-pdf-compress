# Merge PDF 技术方案

## 目标

在现有 PDF 压缩工具的基础上，增加第一个矩阵工具：

- `Merge PDF`

目标不是做复杂编辑器，而是先做一个简单、清晰、可上线的 PDF 合并工具。

## 一、产品目标

用户可以：

1. 上传多个 PDF
2. 调整文件顺序
3. 点击合并
4. 下载一个合并后的 PDF

第一版不做：

- 登录
- 云端保存
- 历史记录
- 单页级拖拽排序
- 页面预览缩略图

## 二、技术方案

当前项目已经使用：

- `Next.js`
- `TypeScript`
- `pdf-lib`

所以 `Merge PDF` 第一版继续沿用浏览器内处理，不引入后端服务。

### 合并方式

使用 `pdf-lib`：

1. 读取多个 PDF 文件
2. 创建新的 `PDFDocument`
3. 逐个 copy pages
4. 按文件顺序 addPage
5. 导出合并后的 PDF

## 三、功能范围

### 第一版范围

- 多文件上传
- 文件列表展示
- 上下移动顺序
- 删除某个文件
- 合并按钮
- 下载合并后的 PDF

### 第二版再考虑

- 拖拽排序
- 页面数显示
- 每个文件页数统计
- 合并前后文件信息

## 四、建议文件结构

建议新增：

- `lib/pdf/merge.ts`
- `components/merge-pdf-card.tsx`
- `app/merge-pdf/page.tsx`

## 五、UI 结构

建议页面结构：

1. 标题区
- `Merge PDF files`
- 简短说明

2. 上传区
- 拖拽 / 选择多个 PDF

3. 文件列表
- 文件名
- 文件大小
- 上移
- 下移
- 删除

4. 主操作区
- `Merge PDF files`
- 下载结果

## 六、状态设计

建议状态：

- `files[]`
- `isMerging`
- `mergedFile`
- `error`

其中 `files[]` 每项包含：

- `id`
- `file`
- `name`
- `size`

## 七、风险

### 1. 大文件合并时浏览器内存压力

第一版接受这个限制。

### 2. 非法或损坏 PDF

要给出清晰错误提示。

### 3. 文件很多时列表过长

列表区域要做最大高度和滚动。

## 八、开发顺序

1. 实现 `merge.ts`
2. 做 `merge-pdf-card.tsx`
3. 新建页面路由
4. 接入导航
5. 补页面 metadata
6. 验证下载流程

## 九、验收标准

- 可以上传多个 PDF
- 可以调整文件顺序
- 可以删除文件
- 可以输出一个合并后的 PDF
- 可以成功下载
- 页面可独立访问

## 十、结论

严格按产品规划推进，下一项就做：

- `P3.1 Merge PDF`

而且应坚持：

- 第一版简单可用
- 不做过重交互
- 先跑通主流程，再考虑拖拽排序和更多高级能力
