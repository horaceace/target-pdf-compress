# 发布检查清单

更新日期：2026-05-31

## 当前发布结论

普通产品版本可以发布；外部强压引擎不能作为生产能力发布。

部署交接见：`发布交接.md`。

开源准备见：`开源准备清单.md`。

## 普通产品发布

发布范围：

- 浏览器端 PDF 压缩
- `Scanned PDF` 浏览器压缩路径
- 上传队列、目标大小、失败后的下一步提示
- SEO 页面、GA4 事件、benchmark 和样本工具链
- 产品规划和 GitHub 开源复用调研

发布前运行：

```bash
npm run release:standard-check
```

该命令会顺序执行：

- `npm run test:compression-strategy`
- `npm run benchmark:pipeline`
- `npm run release:readiness`
- `npm run build`
- `npm run release:seo-check`
- `npm run cf:build`

通过标准：

- 命令退出码为 `0`
- `release:readiness` 中 `Standard Release` 为 `PASS`
- `npm run build` 成功生成页面
- `npm run release:seo-check` 确认 robots 禁止 `/dev/` 且 sitemap 不含开发路径
- `npm run cf:build` 成功生成 Cloudflare Workers bundle

## CI 发布检查

GitHub Actions 使用：

```bash
npm run release:ci-check
```

它会从 fresh clone 可复现路径执行：

- `npm run fixtures:compression`
- `npm run test:compression-strategy`
- `npm run benchmark:compression`
- `npm run build`
- `npm run release:seo-check`
- `npm run cf:build`

说明：

- CI 检查不依赖 `test-fixtures/pdf-compression/results/*`
- CI 检查不依赖真实 PDF 样本
- CI 检查不代表强压引擎可发布

## 发布后检查

部署完成后运行：

```bash
npm run release:postdeploy-check
```

默认检查 `https://filesmaller.space`。可用 `RELEASE_BASE_URL` 指向 Workers 预览域名。

检查范围：

- 首页 HTTP 200
- `/compress-pdf` HTTP 200
- `/robots.txt` 包含 `Disallow: /dev/`
- `/sitemap.xml` 包含生产压缩页且不包含 `/dev/`
- `/dev/compression-benchmark` HTTP 404

`release:readiness` 会检查这个命令是否存在，但不会在部署前执行公网检查。

## 开源发布检查

公开 GitHub 仓库前运行：

```bash
npm run release:open-source-check
```

通过标准：

- `README.md` 存在
- `CONTRIBUTING.md` 存在
- `CHANGELOG.md` 存在
- `LICENSE` 或 `LICENSE.md` 已提交
- `package.json.license` 已声明
- `package.json.private` 已改为 `false`
- 真实 PDF 仍被 git 忽略
- benchmark results 仍被 git 忽略

当前阻塞：

- 未选择 License
- `package.json.license` 缺失
- `package.json.private` 仍为 `true`

说明：

- 普通产品发布通过，不等于仓库可以公开开源。
- 强压引擎阻塞，不影响普通产品上线，但会影响开源 README 中的宣传口径。

## 强压引擎发布

以下内容当前不能发布为生产能力：

- qpdf 生产接入
- pdfcpu 生产接入
- Ghostscript 生产接入
- server-side strong compression
- 保证达到严格目标大小

强压发布必须满足：

- 真实样本 `8/8`
- 浏览器 benchmark 有真实样本行
- 人工画质复核无 `TBD`
- 外部引擎有真实样本成功结果
- `npm run benchmark:decision-gate` 为 `PASS`
- `npm run benchmark:engine-recommendation` 给出推荐引擎

当前阻塞：

- 真实样本 `0/8`
- 强压决策门禁 `BLOCKED`
- 引擎推荐 `BLOCKED`
- 人工画质复核仍有 `TBD`
