## 发布前本地检查

先查看统一进度：

```bash
npm run project:status
```

该命令会刷新普通发布、强压缩、开源准备三类状态报告，并输出整体百分比。

运行：

```bash
npm run release:standard-check
```

必须看到：

```text
Standard release check passed.
Current browser-first FileSmaller product changes are releasable.
```

该命令会检查：

- 压缩策略测试
- 压缩证据流水线
- 普通发布就绪状态
- Next.js production build
- robots / sitemap SEO 检查
- Cloudflare Workers build

CI 使用：

```bash
npm run release:ci-check
```

CI 检查只覆盖 fresh clone 可复现部分，不替代真实样本和强压引擎门禁。

## 部署方式

### 推荐方式：推送 master 自动部署

GitHub Actions 文件：

```text
.github/workflows/deploy-cloudflare.yml
```

触发条件：

- push 到 `master`
- 手动 `workflow_dispatch`

CI 流程：

```text
npm ci
npm run release:ci-check
npm run cf:deploy
```

`release:ci-check` 会在 fresh clone 环境生成 synthetic fixtures，并验证策略测试、Node benchmark、Next.js build、robots / sitemap SEO 检查和 Cloudflare Workers build。它不依赖本地 ignored 的真实样本或 results 文件。

必需 GitHub Secrets：

- `CLOUDFLARE_API_TOKEN`
- `CLOUDFLARE_ACCOUNT_ID`

### 本地部署方式

仅在确认 Cloudflare 登录和权限无误时使用：

```bash
npm run cf:deploy
```

本地部署前仍需先跑：

```bash
npm run release:standard-check
```


## 发布口径

可以说：

- FileSmaller 是 browser-first PDF compression 工具
- 文件在浏览器中处理
- 支持 PDF 压缩、合并、拆分、旋转、页面删除/重排、PDF/JPG 转换
- 针对扫描件和图片型 PDF 提供 `Scanned PDF` 浏览器压缩模式
- 压缩结果取决于文件内容

不能说：

- 已上线 qpdf 强压
- 已上线 pdfcpu 强压
- 已上线 Ghostscript 强压
- 服务端强压已验证
- 可以保证压到指定大小

## 开源交接

当前 GitHub 仓库不建议直接公开为开源项目。

公开前必须先处理：

- 选择 License，建议默认 `MIT`，如更重视专利条款可选 `Apache-2.0`
- 新增 `LICENSE` 或 `LICENSE.md`
- 在 `package.json` 中写入对应 `license`
- 将 `package.json.private` 改为 `false`
- 运行并通过 `npm run release:open-source-check`

开源后仍不能把强压缩描述为生产能力。README 和发布口径应继续保持 browser-first 定位。
